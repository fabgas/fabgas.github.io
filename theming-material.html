Le but de ce document est de décrire comment appliquer un thème à un composant material. Initialement, ce composant aura le thème standard et il s’agit d’atteindre un nouveau style.

Composant standard

Le composant choisi comme cible sera le bouton : mat-button.

Après avoir construit un projet Angular à l’aide de ng new, l’exemple des boutons du site material est pris comme composant de base. L’aspect de l’application est alors le suivant

Les buts est de comprendre comment modifier certaines propriétés de ce composant/

  1. modifier les couleurs standard

  2. modifier le style du bouton (bordure, font)

Jouer avec les thèmes

Modifier globalement la palette

Pour modifier les couleurs globalement (et donc également pour un bouton de base), il faut redéfinir la palette. En material / saas, la palette est définie dans une map qui contient les teintes de la palette et une sous-map contient la couleur de contraste pour chacune des teintes. La palette de contraste sert à définir la couleur du texte lorsque la couleur de base est utilisée en background.

Exemple de palette
$indigo-palette: (
 50: #e8eaf6,
 100: #c5cae9,
 200: #9fa8da,
 300: #7986cb,
 // ... continues to 900
 contrast: (
   50: rgba(black, 0.87),
   100: rgba(black, 0.87),
   200: rgba(black, 0.87),
   300: white,
   // ... continues to 900
 )
);

Les couleurs, palettes sont définis dans le fichier _palette.scss se trouvant dans @angular/material/core/theming

On peut définir notre propre palette. Pour cela il faut construire une map contenant une valeur pour 50 et chaque centaine entre 100 et 900, soit dix couleurs. Il faut y ajouter la map de contraste.

Pour s’aider, il est possible d’utiliser cet outil :https://material.io/design/color/the-color-system.html#tools-for-picking-colors

Définition d’un thème

Pour définir son propre theme, il faut créer un fichier _theme.scss

_themes.scss
@import '~@angular/material/theming'; (1)
@include mat-core();

$my-app-primary: mat-palette($mat-blue-grey); (2)
$my-app-accent:  mat-palette($mat-pink, 500, 900, A100); (3)
$my-app-warn:    mat-palette($mat-deep-orange);(4)
$my-app-theme: mat-light-theme($my-app-primary, $my-app-accent, $my-app-warn); (5)

@include all-component-themes($my-app-theme);(6)
1 Import du thème d’angular material
2 Redéfinition de la couleur "primary". $mat-blue-grey correspond à la palette blue-grey
3 Redéfinition de la couleur "accent"
4 Redéfinition de la couleur "warn"
5 Création de la variable theme en reprenant les trois couleurs précédentes
6 appel de la fonction angular-material-theme

et déclarer ce fichier dans la liste des styles .

angular.json
"styles": [
  "styles.css",
  "_themes.scss"
]

Ajouter plusieurs thèmes

Il est possible de créer des thèmes alternatif en imbriquant une définition de thème dans une classe à un élément parent pour que le thème alternatif prenne effet.

Chaque variable se voit attribuer une couleur. L’utilisation de la fonction mat-palette permet d’utiliser les palettes définies par angular material. Elle prend en premier argument un nom de couleur, les trois autres argument étant la valeur par default, une valeur plus brillante et une valeur sombre.

_themes.scss
@import '~@angular/material/theming';
@include mat-core();
$my-app-primary: mat-palette($mat-blue-grey);
$my-app-accent:  mat-palette($mat-pink, 500, 900, A100);
$my-app-warn:    mat-palette($mat-deep-orange);
$my-app-theme:   mat-light-theme($my-app-primary, $my-app-accent, $my-app-warn);
@include all-component-themes($my-app-theme);

.alternate-theme { (1)
  $alternate-primary: mat-palette($mat-light-blue);
  $alternate-accent:  mat-palette($mat-yellow, 400);
  $alternate-theme: mat-light-theme($alternate-primary, $alternate-accent);
@include  all-component-themes($alternate-theme);(2)
}
dans les précédentes versions d’angular material (<13) la fonction utilisée était angular-material-theme, dorénavant il semblerait que ce soit all-component-themes.
1 Définition de la classe qui va contenir le thème alternatif. <2>Ajout du thème dans la liste des thèmes.

il suffit d’indiquer le nom de la classe dans le html, sur un tag, pour quelle soit utilisée.

<section class="alternate-theme">
  <div class="example-label">Flat</div>
  <div class="example-button-row">
    <button mat-flat-button>Basic</button>
    <button mat-flat-button color="primary">Primary</button>
    <button mat-flat-button color="accent">Accent</button>
    <button mat-flat-button color="warn">Warn</button>
    <button mat-flat-button disabled>Disabled</button>
    <a mat-flat-button href="https://www.google.com/" target="_blank">Link</a>
  </div>
</section>

Les counleurs sont prises dans des palettes. Dans la ligne

 $alternate-primary: mat-palette($mat-light-blue);

$mat-light-blue est une des palettes de material angular. La liste des palettes est disponibles dans le fichier _palette.scss (@angular/material/core)

Modification de l’aspect du bouton

https://betterprogramming.pub/best-way-to-overwrite-angular-materials-styles-e38dc8b84962

L’objectif de cette partie est de modifier l’aspect du bouton.

Pour comprendre le styling du bouton, il faut analyser le style par la console de debug du navigateur. Dans notre cas, on va chercher à modifier la bordure du bouton. Il faut donc trouver la classe ou la combinaison de classes qui affecte cette partie du bouton.

La console montre que la classes qui impacte le style du bouton est .mat-stroked-button

Il faut donc être en mesure de surcharger cette classe en utilisant sass.

Pour cela, il faut :

  1. créer un fichier "_button.scss" et y construire le style

  2. importer ce fichier dans styles.scss

  3. appliquer la classe au bouton

Pour le point 1, une nouvelle classe my-mat-button permettra d’appliquer spécifiquement ce style. En créant une classe, le style pourra s’appliquer plus précisément à chacun des composants.

_my-button.scss
.my-mat-button {
    &.mat-stroked-button {
      border: 2px solid black;
      border-radius: 10px;
  }
}

Pour que cette classe puisse être accessible à l’ensemble de l’application, le fichier my-button.scssest importé dans le fichier styles.scsss_.

@import "<<chemin>>/my_button.scss";

Enfin dans le composant, le tag cible se voit attribuer la classe "my-mat-button":

 <button  mat-stroked-button   class="my-mat-button" >
    <mat-icon>home</mat-icon>
  </button>

A gauche le bouton material et à droite le bouton avec le nouveau style.

A l’aide de SASS i lest possible d’impacter plusieurs niveaux dans le DOM. En ajoutant des règles pour la même classe, diverses combinaisons peuvent surcharger le style à l’aide d’une seule classe. La partie html est non modifiée.

.my-mat-button {
    &.mat-stroked-button {
      border: 2px solid black;
      border-radius: 10px;
  }
   &.mat-stroked-button{
     cursor: crosshair;
   }
}
Dans la console, lorsque les classes sont séparées par des "," il s’agit de combinaisons différentes.

Le ciblage de la bonne combinaison dépend de là ou est positionnée la classe. Dans ce cas il faut paramétrer les 4 classes pour atteindre le bon niveau lorsque la classe est postionnée au niveau du mat-form-field.

Collé ou not collé ? lorsque dans la déclaration sass deux noms de classes sont collés cela signifie que les deux classes doivent se situer au même niveau dans la hiérarchie.

Dans la définition des rules, il est possible d’utiliser les mixins et fonctions définies préalablement

.my-mat-select {
  &.mat-form-field .mat-form-field-wrapper .mat-form-field-flex .mat-form-field-infix {
   margin-top: -0.25em;
   border : 2px solid mat-color($my-app-primary);
   border-radius : 8px;
  }
}

Pour appliquer globalement certains styles, il suffit d’enlever la classe de la règle

.my-tab-class {
  &.mat-tab-group.mat-primary .mat-ink-bar {
    background-color: mat-color($my-app-primary);
    height: 4px;
  }
}

devient

.mat-tab-group.mat-primary .mat-ink-bar {
    background-color: mat-color($my-app-primary);
    height: 4px;
  }

Sass

Avant d’aller plus loin il faut s’arrêter sur les notions propre à SASS tels que la mixin et les fonctions.

Mixin

https://www.ice-dev.com/decoupe-html-css/sass-lecon-4-les-mixins-et-les-fonctions/

Une mixin est un morceau de code css, qui peut être paramétrable, réutilisable dans un fichier SASS (scss).Une mixin accèpe des paramètres en entrée (une couleur, une valeur,…​).

Pour définir une mixin il faut utiliser la directive @mixin et lui donner un nom . Ensuite, dans un fichier SASS (du composant par exemple) , la directive @include permet de l’utiliser.

Dans un fichier sass.
@mixin my_mixin {
 // css code
}
// dans le fichier sass.
@include my_mixin

Pour la mise en oeuvre des paramètre, il suffit de les lister dans à la suite du nom de la mixin. Une valeur par défaut peut être définie à la suite du paramètre .

@mixin my_mixin($param1, $param2 : default)

@include my_mixin($value1,$value2)
@include my_mixin($value1) , value2 vaut default
Exemple concret
@mixin my_mixin($align, $margin : 10px) {
    display: inline-block;
    vertical-align : $align;
    margin-left: $margin
}
// utilisation
.container {
    @include inline-block(middle)
}

Les fonctions

La différence entre une mixin et une fonction est le résultat obtenu . La mixin permet d’obtenir du code, la fonction retourne une valeur.

Grâce aux fonctions, il sera possible de calculer des valeurs tels que des %, concertir des unités, changer une couleur …​

@function percentage($num) {
    // code
}

.bloc {
    width: percentage($num/$container-width);
}

Conditions et boucles

Tel un langage évolué, SASS dispose de mots clés pour gérer les contions

$color : blue;
$border :'';

@if ($color == red) {
    $border : 2 solid black
}
@elseif ($color == blue) {
    $border : 2 solid red
}
@else {
    $border : none
}

.select {
    color: $color;
    border: $border;
}

La notion de boucle est aussi présente en SASS.

boucle for pour construire des class grid-1,grid-2,…​
@for $i from 1 through 6 {
    .grid-#{$i} {
        width: 100px*$i;
    }
}
utilisation de while
$i: 6;

@while $i > 0 {
    .item-#{$i} { width: 2em * $i; }
    $i: $i - 2;
}

et enfin un for each

@each $animal in puma, sea-slug, egret, salamander {
    .#{$animal}-icon {
      background-image: url('/images/#{$animal}.png');
    }
}

Imbrication des règles

L’imbrication des règles permet de rapidement créer des classes pour les différents états css.

$link_color: #ce649b;
$color_hover: darken($link_color, 20%); // fonction SASS qui assombrit la couleur
$color_focus: lighten($link_color, 10%); // fonction SASS qui éclaircit la couleur

// & : représente l'élément parent

.my_class a {
     color: $link_color;
     &:hover {
         color: $color_hover;
     }
     &:focus {
         color: $color_focus;
     }
}

Ce code va permettre de créer trois classes (my_class a , my_class a:hover, my_class a:focus)
